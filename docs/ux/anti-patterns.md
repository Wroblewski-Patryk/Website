# UX Anti-Patterns

Use this file to block recurring design mistakes before they become normalized.

## Layout Anti-Patterns

- Treating desktop as centered mobile content with unused empty space.
- Treating tablet as a stretched phone with no extra utility.
- Filling screens with identical cards that all compete equally for attention.
- Using too many nested containers, borders, and panels without hierarchy gain.

## Visual Anti-Patterns

- Generic SaaS styling with no visual thesis.
- Overusing gradients, blur, glass, or glow without product purpose.
- Replacing approved painterly or illustrated backgrounds with cheap gradient
  approximations.
- Making everything prominent so nothing feels important.
- Relying on color alone to create hierarchy.

## Interaction Anti-Patterns

- Hidden primary actions.
- Important controls placed too far from the user's current context.
- Motion that delays interaction or makes the UI feel heavy.
- Hover-dependent understanding for touch-first flows.

## Responsive Anti-Patterns

- Copying one layout unchanged across all breakpoints.
- Shrinking elements instead of restructuring them.
- Ignoring landscape tablet and narrow desktop states.
- Preserving desktop density on touch surfaces without comfort checks.

## State Anti-Patterns

- Spinner-only loading with no structure hint.
- Empty states that explain nothing and offer no next step.
- Errors that blame the user or hide recovery actions.
- Success feedback that is too subtle to trust or too loud to tolerate.

## Delivery Anti-Patterns

- Treating a canonical screenshot as inspiration instead of a spec.
- Skipping a visual gap audit before implementation.
- Collapsing layout, asset prep, and parity review into one vague task.
- Calling a screen "done" without side-by-side screenshot comparison.

## Accessibility Anti-Patterns

- Low-contrast text for the sake of aesthetics.
- Tiny targets and cramped controls.
- Invisible or weak focus treatment.
- Interfaces that break when users zoom or increase text size.
